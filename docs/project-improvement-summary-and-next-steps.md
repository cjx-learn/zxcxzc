# mall-swarm 项目改造说明与后续计划

## 1. 项目改造目标

本项目基于 `macrozheng/mall-swarm` 电商系统进行二次开发，课程作业主题为：

```text
基于电商用户行为数据的商品推荐与用户行为分析系统设计与实现
```

改造目标不是只展示原商城功能，而是在原有商品、用户、订单等业务基础上，增加用户行为采集、用户画像、商品画像、推荐算法、数据分析看板和用户端推荐展示能力。

## 3. 已完成的主要改进

### 3.1 云服务器部署

已完成 mall-swarm 项目在云服务器上的基础部署，包括 MySQL、Redis、MongoDB、RabbitMQ、Nacos、MinIO、Nginx 以及主要后端服务。

当前系统可以通过浏览器访问管理端、用户端和分析看板，满足课程作业演示要求。

### 3.2 用户行为数据体系

系统已围绕用户行为分析增加行为事件数据能力，核心行为包括：

| 行为类型 | 含义 | 分析价值 |
| --- | --- | --- |
| `view` | 浏览商品 | 表示用户曝光和初步兴趣 |
| `search` | 搜索商品 | 表示用户主动需求 |
| `fav` | 收藏商品 | 表示较强兴趣 |
| `cart` | 加入购物车 | 表示购买意向 |
| `order` | 下单 | 表示转化行为 |
| `pay` | 支付 | 表示最终购买行为 |

这些行为进入统一的用户行为事件表，后续用户画像、商品画像、热门推荐和个性化推荐都基于行为数据计算。

当前已补充 HTTP 行为上报接口：

```text
POST /mall-analytics/analytics/event
```

该接口支持用户端或其他业务服务直接上报 `view/search/fav/cart/order/pay` 行为，并写入 `user_behavior_event` 表。后续只需要在 H5 页面关键操作处调用该接口，就可以形成更完整的真实行为采集链路。

当前用户端 H5 首页商品卡片点击已接入 `view` 行为上报，点击推荐商品、新品、热门商品、秒杀商品时会异步写入行为事件，不阻塞页面跳转。

### 3.3 用户画像分析

已实现全体用户画像和单用户画像分离展示。

全体用户画像展示：

- 画像用户数
- 高价值用户数
- 潜在购买用户数
- 平均活跃天数
- 用户等级分布
- 偏好分类分布
- RFM 活跃用户排行

单用户画像展示：

- 用户基础信息
- RFM 综合分
- R、F、M 分项得分
- 最近活跃时间
- 偏好分类
- 用户行为汇总
- 个性化推荐商品

### 3.4 活跃用户 TOP 与 RFM 模型

左侧用户选择栏已由“手动输入用户 ID”改为“活跃用户 TOP 列表”。

用户按 RFM 综合分从高到低排列，点击用户即可切换到该用户的个人画像。

RFM 计算口径：

```text
RFM = R * 40% + F * 40% + M * 20%
```

| 指标 | 含义 |
| --- | --- |
| R Recency | 最近活跃时间，越近分越高 |
| F Frequency | 行为频率，行为越多分越高 |
| M Monetary | 消费价值，支付金额越高分越高 |

左侧还增加了用户关键词、RFM 等级、最低 RFM 分等筛选条件。

### 3.5 商品画像与商品分析

商品分析模块已支持：

- 商品 ID 查询
- 商品名称关键词查询
- 商品货号查询
- 无商品时显示“无商品”
- 单商品行为汇总
- 商品热度分
- 加购率、下单率、支付率
- 同分类相似商品推荐

这使商品分析不再局限于输入 ID，演示时可以通过关键词搜索商品，再点击商品卡片查看分析。

### 3.6 推荐结果展示优化

推荐结果模块已完成多次优化：

- 推荐卡片增加 TOP 排名
- 商品图片、名称、价格、推荐类型、推荐原因分区展示
- 修复推荐原因中文乱码问题
- 默认展示 DeepFM 推荐
- 增加算法切换栏
- 不再把 DeepFM、NCF、ItemCF、规则推荐混在一个列表中

当前推荐结果页支持切换：

```text
DeepFM推荐 / NCF推荐 / ItemCF推荐 / 规则推荐
```

这样可以避免不同算法的推荐分混在一起比较，逻辑更严谨。

### 3.7 推荐算法扩展

除原有规则推荐和热门推荐外，已增加基于淘宝用户行为数据训练的推荐算法。

| 算法 | 当前状态 | 分数含义 |
| --- | --- | --- |
| 热门推荐 | 已实现 | 商品行为热度分 |
| 规则推荐 | 已实现 | 偏好分类下的商品热度分 |
| ItemCF | 已训练并导入 | 商品协同相似度强度 |
| NCF | 已训练并导入 | 用户与商品匹配概率 |
| DeepFM | 已训练并导入 | 综合特征交叉后的兴趣概率 |

DeepFM 当前作为主推荐算法，NCF 和 ItemCF 作为对比算法。

### 3.8 深度学习推荐训练流程

已建立 `ml-recommend` 模块，用于离线训练和导出推荐结果。

已完成脚本包括：

| 脚本 | 作用 |
| --- | --- |
| `sample_taobao.py` | 从淘宝行为数据集中抽样 |
| `train_itemcf.py` | 训练 ItemCF 推荐 |
| `train_ncf.py` | 训练 NCF 神经协同过滤 |
| `train_deepfm.py` | 训练 DeepFM 特征交叉模型 |
| `export_recommend_sql.py` | 将模型推荐结果导出为 SQL |
| `evaluate_recommendations.py` | 计算 HitRate@K、Precision@K、Recall@K、NDCG@K 等评估指标 |
| `run_recommend_pipeline.py` | 串联训练、评估、SQL 导出的离线推荐流水线 |

已使用淘宝用户行为数据集进行抽样训练，并将 DeepFM、NCF、ItemCF 推荐结果导入系统数据库。

当前推荐训练已采用时间切分方式：

```text
每个用户前 80% 行为 -> 训练集
每个用户后 20% 行为 -> 测试集
```

这样可以避免用全量历史训练后再评估导致的数据泄漏问题，也能让推荐指标更接近真实离线评估。

### 3.9 推荐算法评估与看板展示

已补充推荐算法离线评估流程，并在分析看板中新增“算法评估”页面。

当前评估方式：

- 使用淘宝行为数据抽样结果进行时间切分。
- 训练期数据用于生成推荐结果。
- 测试期的收藏、加购、支付、购买、下单行为作为正样本。
- 对 DeepFM、NCF、ItemCF、热门推荐进行统一 TopK 评估。

已实现评估指标包括：

| 指标 | 含义 | 展示价值 |
| --- | --- | --- |
| HitRate@K | 用户测试期真实感兴趣商品是否出现在推荐 TopK 中 | 衡量是否命中真实商品 |
| Precision@K | TopK 推荐中命中的比例 | 衡量推荐列表准确性 |
| Recall@K | 测试期正样本被推荐覆盖的比例 | 衡量召回能力 |
| NDCG@K | 命中商品越靠前得分越高 | 衡量排序质量 |
| 分类命中率 | 推荐商品分类是否命中用户测试期兴趣分类 | 更适合课程演示解释兴趣匹配 |
| 覆盖率 | 推荐结果覆盖的商品比例 | 衡量推荐多样性 |

由于淘宝外部商品会映射到 mall-swarm 演示商品，商品 ID 级命中较稀疏，因此答辩展示时建议重点解释“分类命中率”和“分类 Precision@K”，再补充说明 Precision@K、Recall@K、NDCG@K 是推荐系统常见标准指标。

已部署接口：

```text
GET /mall-recommend/recommend/evaluate
```

看板页面已展示评估用户数、最佳分类命中算法、最佳 NDCG 算法、指标对比柱状图和算法评估明细表。

- 定时训练

- 推荐结果版本管理

### 5.5 已修复：旧商品图片历史数据问题
